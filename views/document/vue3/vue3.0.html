# 2.ref函数

RefImpl => 引用实现对象 : ref => reference => 引用, impl => implement => 实现

  • 作用：定义一个响应式的数据
  • 语法：const xxx = ref(initValue)
    • 创建一个包含响应式数据的引用对象（reference对象，简称ref对象）。
    • JS中操作数据：xxx.value
    • 模板中读取数据：不需要.value，直接：<div></div>
  • 备注：
    • 接受的数据可以是：基本类型、也可以是对象类型。
    • 基本类型的数据：响应式依然是靠object.defineProperty()的get与set完成的。
    • 对象类型的数据：内部**”求助“**了Vue3.0中的一个新函数——reactive函数（proxy是封装再reactive函数中，所以原理还是proxy）。

# 3.reactive函数

  • 作用：定义一个对象类型的响应式数据（基本类型不能用它，要用ref函数）

    • 语法：const 代理对象 = reactive（源对象）接受一个对象（或数组），返回一个代理对象(Proxy的实例对象，简称proxy对象)
    • reactive定义的响应式数据式“深层次的”
    • 内部基于ES6的Proxy实现，通过代理对象操作源对象内部数据进行操作
    import { reactive } = 'vue'
let person = reactive({
    name: '王五',

    job: {
        type: '全栈工程师'

        person.hobby[0] = '魔术'

function changePerson() {
    person.age = 19
    person.job.type = '所以你是小丑我是大师'
    person.hobby[0] = '魔术'
}
return {
    person,

# 4.reactive对比ref

  • 从定义数据角度对比：
    • ref用来定义：基本类型数据
    • reactive用来定义：对象（或数组）类型的数据
    • 备注：ref也可以用来定义对象（或数组）类型的数据，它内部会自动通过reactive转为代理对象
  • 从原理角度对比：
    • ref通过Object.defineProperty()的get与set来实现响应式roxy（数据劫持）
    • reactive通过使用Proxy来实现响应式（数据劫持），并通过Reflect操作源对象内部的数据
  • 从使用角度对比：
    • ref定义的数据：操作数据需要.value,读取数据时模板（templete）中直接读取不需要.value
    • reactive定义的数据：操作数据与读取数据均不需要.vaiue

# Vue3.0的响应式

  • 实现原理：

    • 通过Proxy(代理)：拦截对象中任意属性的变化，包括：属性值的读写、属性的添加、属性的删除等

    • 通过Reflect(反射)：对源对象的属性进行操作

    • MDN文档中描述的Proxy与Reflect:

      • Proxy: https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Proxy (opens new window)

      • Reflect:https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Reflect (opens new window)

        new Proxy(data, {
    // 拦截读取属性值
    get(target, prop) {
        return Reflect.get(target, prop)
    },
    // 拦截设置属性值或添加新属性
    set(target, prop, value) {
        return Reflect.set(target, prop, value)
    },
    // 拦截删除属性
    deleteProperty(target, prop) {
        return Reflect.deleteProperty(target, prop)
    }
})

# 1.comouted函数

  • 与Vue2.x中的computed配置功能一致

  • 写法:

    let {computed} from 'vue'

setup() {
    let person = reactive({
        firstName: '张'，
        lastName: '三'
    })
    // 简写方式
    let fullName = computed(() => {
        return `${person.firstName}-${person.lastName}`
    })
    // 完整方式
    let fullName = computed({
        get() {
           return `${person.firstName}-${person.lastName}` 
        },
        set(value) {
            let nameArr = value.split('-')
            person.firstName = nameArr[0]
            person.lastName = nameArr[1]
        }
    })
    return {
        person,
        fullName
    }
}

# 2.watch函数

  • 与Vue2.x中的watch配置功能一致
    • immediate: true，初始触发配置，记住这个单词
  • 两个小“坑”：
    • 监视reactive定义的响应式数据时：oldVllue无法正确获取
    • 监视reactive定义的响应式数据时：强制开启了深度监视（deep配置失效）
    • 监视reactive定义的响应式数据中某个属性（属性值为引用数据类型）时：deep配置有效
import { watch } from 'vue'
// 情况一：监视ref定义的响应数据（基本数据类型）
watch(sum, (news, old) => {
    console.log(news, old)
}, {immediate: true})
// 情况二：监视多个ref定义的响应数据（基本数据类型）
watch([sum, msg], (news, old) => {
    console.log(news, old)
})
/*
情况三：监视reactive定义的响应式数据（引用数据类型）
	  此处无法正确获取旧数据old值
	  deep:false配置项不再生效
*/
watch(person, (news, old) => {
    console.log(news, old)
},{immediate: true, deep: false})
// 情况四：监视reactive定义的响应式数据中的某个属性（此属性为基本数据类型）
watch(() => person.name, (news, old) => {
    console.log(news, old)
})
// 情况五：监视reactive定义的响应式数据中的多个属性（此属性为基本数据类型）
watch([_ => person.name, _ => person.age], (news, old) => {
    console.log(news, old)
})
/*
情况六：监视reactive定义的响应式数据中的多个属性（此属性为引用数据类型）
	  这种情况deep:true才能用上派场
*/
watch([_ => person.obj], (news, old) => {
    console.log(news, old)
}, {deep: true})

# 3.watchEffect函数

  • watch的套路是：既要指明监视的属性，也要指明监视的回调
  • watchEffect的套路是：不用指明监视哪个属性，监视的回调中用到哪个属性，就监视哪个属性
  • wacthEffct有点像computed:
    • 但computed注重的计算出来的值（回调函数的返回值），所以必须要写返回值
    • 而watchEffect更注重的是过程（回调函数的函数体），所以不用写返回值
import {watchEffect} from 'vue'
watchEffect(() => {
    const x1 = sum.value
    const x2 = person.age
    console.log('watchEffect配置的回调执行了')
})

# 8.生命周期

  • vue3中可以继续使用vue2中的声明周期钩子，但是有两个被更名：
    • beforeDestroy改名为beforeUnmount
    • destroyed改名为unmounted
  • vue3也提供了Composition API 形式的生命周期钩子，与vue2中钩子对应关系如下：
    • beforeCreat=====>setup()
    • created========>setup()
    • beforeMount=====>onBeforeMount
    • mounted========>onMounted
    • beforeUpdate====>onbeforeUpdate
    • updated========>onUpdated
    • beforeUnmount====>onBeforeUnmount
    • unmounted=======>onUnmounted
  • 冷知识：
    • vue2若没有进行vm.$mount(el),也会走前beforeCreat、created声明周期
    • 而vue3如果没有app.mount(el)，则不会进入生命周期的环节

# 9.自定义hook函数

  • 什么是hook？——本质是一个函数，把setup函数中使用的Composition API进行了封装
  • 类似与vue2中的mixin
  • 自定义hook的优势:复用代码，让setup中的逻辑更清楚易懂

例子：

hooks/usePonits.js文件:

// hooks/usePonits.js文件
import {onMounted, reactive, onBeforeUnmount} from 'vue'
export default function() {
  let point = reactive({
    x: 0,
    y: 0
  })
  function savePoint (event) {
    console.log(event.pageX,event.pageY,'enevt')
    point.x = event.pageX
    point.y = event.pageY
  }
  onMounted(() => {
    window.addEventListener('click', savePoint)
  })
  onBeforeUnmount(() => {
    window.removeEventListener('click', savePoint)
  })
  return point
}

# 10.toRef、toRefs

  • 作用：创建一个ref对象，其value值指向另一个对象中的某个属性
  • 语法：const name = toRef(person, ‘name’)
  • 应用：要将响应式对象中的某个属性单独提供给外部使用时
  • 扩展：toRefs与toRef功能一致，但可以批量创建多个ref对象，语法...toRefs(person)将person对象中的第一层属性转为响应式

# 三、其他Composition API

# 1.shallowReactive与shallowRef

  • shallowReactive: 只处理对象的第一层次属性的响应式（浅响应式）
  • shallowRef:只处理基本数据类型的响应式，不进行对象的响应式处理
  • 什么时候用？
    • 如果有一个对象数据，结构比较深，但变化时只是外层属性变化===>shallowReactive
    • 如果有一个对象数据，后续功能不会修改该对象内部的属性，而是用新的对象来替换===>shallowRef

# 2.readonly与shallowReadonly

  • readonly:让一个响应式数据变为只读（深只读）
  • shallowReadonly:让一个响应式数据变为只读的（浅只读）
  • 应用场景：不希望数据被修改

# 3.toRaw与markRaw

  • toRaw:
    • 作用：将一个由refactive生成的响应式对象转为普通对象
    • 使用场景：用于读取响应式对象转为对应的普通对象，对这个普通对象的所有操作，不会引起页面更新
  • markRaw:
    • 标记一个对象，使其永远不会成为响应式对象
    • 应用场景：
      1. 有些值不应被设置为响应式，例如复杂的第三方库等
      2. 当渲染具有不可变数据的大列表时，跳过响应式转化可以提高性能

# 4.customRef

  • 作用：创建一个自定义的ref,并对其依赖项跟踪和更新触发进行显示控制
import {customRef} from 'vue'
setup() {
    function myRef(value,delay) {
      let timer
      return customRef((track, trigger) =>{ // track：追踪; trigger:触发
        return {
          get() {
              track()
              return value
          },
          set(newValue) {
            clearTimeout(timer)
            timer = setTimeout(() => {
                trigger()
              value = newValue
            }, delay)
          }
        }
      })
    }
    let keyWord = myRef('hello', 500)
    return {keyWord}
  }

      Vue.mixin app.mixin 混入
      Vue.use app.use 使用中间件

# 2.Fragment

不重要，面试被问知道即可

  • 在vue2中：组件必须有一个根标签
  • 在vue3中：组件可以没有根标签，内部会将多个标签包含在一个Fragment虚拟元素中
  • 好处：减少标签层级，减小内存占用
