# 使用

通过 npm引入：

$ npm i fair-wing
1

vue注册：

// main.js文件中
Vue.use(Fair)
1
2

使用说明：

为了避免和vue内置方法或其他插件及公共方法名称冲突,在注册插件向vue上绑定方法时，将该插件的名称进行了特殊的区分 ，例如table()方法需要在其前面添加w-

// <table/>应写为
<w-table/>
1
2

# 单独引用

若不想在vue上注册成全局组件方法，也可以在组件内单独引用直接使用

// main.js文件中
Vue.use(Fair,
{
  Views: {
    bindVue: false
  }
})

# Form

props：

  • fieldModel：table整体的描述
  • fieldList：每列的描述（包括每列的单元格中的内容描述）
  • formData：form数据，相当于model

  • 对表格整体样式的描述,并做了一些少量的默认设置
  • 该表格中的field代表fieldList的每一项（item）

# Select Events

参数 说明 参数 备注
change 选中值发生变化时触发 {value, field, unit, data}
value:目前的选中值
field:当前列配置，Object
unit:列表对应当前值的对象，Object
data:所有数据，Object
visibleChange 下拉框出现/隐藏时触发 {status,field,data}
status:目前的选中值，布尔值
field:当前列配置，Object
data:所有数据，Object		 —
removeTag 多选模式下移除tag时触发 {value,field}
value:移除的tag值
field:当前列配置，Object
clear 可清空的单选模式下用户点击清空按钮时触发 { field, data}
field:当前列配置，Object
data:所有数据，数组
blur 当 input 失去焦点时触发 {event, field, data}
event:$event
field:当前列配置，Object
data:所有数据，数组
focus 当 input 获得焦点时触发 {event, field, data}
event:$event
field:当前列配置，Object
data:所有数据，数组

# v-slot:header

v-slot:header={field,data,model,extra}插槽把所有field.slotHead不为false或者为空的列全部作用，

但也可通过作用于参数做不同的展示

<template v-slot:header={field,data,model,extra}>
	{{field.label}}slot
</template>

# 单元格插槽

单元格插槽当field.type=‘slot’时，激活该列单元格插槽

作用域参数介绍，{scope,field,data,model,extra}：

  • scope：当前行数据
  • field：当前列配置
  • data：整体数据
  • model：整体配置（可以通过父组件直接获取）
  • extra：额外数据（辅助数据）

# v-slot:${field['key']}

作用于某一列单元格的插槽

<!-- 
##操作列常用
field.type='slot'
field.key='handle'
-->
<template v-slot:handle={scope,field,data,model,extra}>
	<el-button v-if="!scope.row.update" type="primary" size='mini' @click="edit(scope)">修改</el-button>
	<el-button v-else type="primary" size='mini' @click="save(scope)">保存</el-button>
</template>

# v-slot:${field['key']}In

也是作用于某一列单元格的插槽，并且作用于form-item中，此插件没有预设某种表单类型时候，就可用此插槽。

需要先通过field.isForm或者**(tableList[index].[fieldModel.updateKey]&&fieldModel.isForm)**组合让单元格先成为form作用域，根据使用场景自由搭配

<!-- 
field.type='slot'
field.key='deviceType'
-->
<template v-slot:deviceTypeIn={scope,field,data,model,extra}>
	<el-input value="v-slot:deviceTypeIn"></el-input>
</template>
